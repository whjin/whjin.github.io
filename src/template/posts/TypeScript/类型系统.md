## 类型 `type`

- 类型注解 `let a:string = 'hello'` 类型推断
- 联合类型 `let a:string | number = 'hello'` 类型之间进行 **或** 操作
- 交叉类型 类型之间进行 **与** 操作
- `never`类型 永不存在值的类型
- `any`类型 任意类型 不进行检测
- `unknown`类型 `any`类型对应的安全类型 默认会进行检测
- 类型断言 `a as string`
- 非空断言 `a!.length`
- 数组类型
  - 类型 `[]`：`let a:(number|string)[] = [1,2,3]`
  - `Array`类型：`let a:Array<number> = [1,2,3]`
- 元祖类型 `let arr:[number,string]=[1,'hello']`
- 对象类型 `type A = {name: string, age?: number}`
- 索引签名 `type A = {name: string, age?: number, [index:string]: any}`
- 函数重载和可调用注解
  ```typescript
  type A = {
    (n: number): number;
    m?: string;
  };
  ```
- 枚举类型和 `const`枚举 `const`编译后的优化
  ```typescript
  enum Roles {
    SUPER_ADMIN,
    ADMIN = 3,
    USER,
  }
  enum Roles1 {
    SUPER_ADMIN = 'super_admin',
    ADMIN = 'admin',
    USER = 'user',
  }
  const enum Roles2 {
    SUPER_ADMIN = 'super_admin',
    ADMIN = 'admin',
    USER = 'user',
  }
  ```
- 字面量类型和 `keyof`关键字 `keyof A`
- 类型保护和自定义类型保护 `typeof` `instanceof` `in` 字面量类型
- 泛型

  ```typescript
  type A<T> = T;
  let a: A<string> = 'hello';
  let b: A<number> = 1;

  type MyArray<T> = T[];
  let d: MyArray<number> = [1, 2, 3];
  let e: number[] = [1, 2, 3];

  interface A<T> {
    (n?: T): void;
    default?: T;
  }
  let fn: A<number> = (n) => {};

  class A<T> {
    n!: T;
  }
  let a = new A<string>();
  class B extends A<string> {}
  let b = new B();
  ```

## 接口 `interface`

**接口与类型别名区别**

接口是一系列抽象方法的声明，是一些方法特征的集合。

- 对象类型
- 接口合并
- 接口继承
- 映射类型
