> 最本质的差异：`Webpack` 是 **先打包，后服务** 的 `Bundle` 驱动模式；`Vite` 是 **先服务，按需编译** 的 `ESM` 原生驱动模式。

## 一、开发启动速度

**核心原因**：**Webpack**必须先完成全量打包才能提供服务；`Vite` 利用现代浏览器原生支持的 `ES Module` 特性，把“打包”工作交给了浏览器，服务器只负责按需编译和转发。

<!-- prettier-ignore-start -->
| **维度** | `Webpack` | `Vite` |
| -------- | --------- | ------ |
| **冷启动原理** |从入口文件开始，递归打包 **所有依赖和业务代码**，<br>生成完整 `bundle` 后再启动开发服务器 |1. 仅对第三方依赖做 **预编译**（`eslint`加速）<br>2. 直接启动开发服务器<br>3. 浏览器请求哪个模块，才 **实时编译** 哪个模块|
|**启动耗时**|与项目规模 **正相关**，大型项目可达 `30s-5min`|几乎与项目规模 **无关**，通常 `1-3s`完成启动|
|**二次启动**|依赖缓存优化，但仍需重新打包变更的依赖树|预构建结果强缓存，二次启动几乎瞬时|
<!-- prettier-ignore-end -->

## 二、热更新（`HMR`）

<!-- prettier-ignore-start -->
|**维度**|`Webpack`|`Vite`|
|---|---|---|
|**更新原理**|重新打包 **变更模块及其所有依赖**，替换整个 `chunk`|基于 `ESM` 精确更新 **变更的单个模块**，不影响其他模块|
|**更新耗时**|与项目规模正相关，大型项目可达 `5-10s`|毫秒级，与项目规模完全无关|
|**状态保留**|部分框架需要额外插件支持，容易出现状态丢失|原生支持更好，状态保留稳定|
|**页面刷新**|复杂变更可能触发整页刷新|极少触发整页刷新|
<!-- prettier-ignore-end -->

## 三、生产构建流程

<!-- prettier-ignore-start -->
|**维度**|`Webpack`|`Vite`|
|---|---|---|
|**构建工具**|自身完全全流程打包|开发用 `esbuild` 预构建，生产用 `Rollup` 打包|
|**构建原理**|基于 `CommonJS/ESM` 混合打包，兼容性优先|纯 `ESM` 打包，`Tree Shaking` 更彻底|
|**构建速度**|中等，大型项目耗时较长|比 `Webpack` 快 `2-10` 倍（`esbuild + Rollup` 双重加速）|
|**产物优化**|优化手段丰富，但配置复杂|内置代码分割、压缩、懒加载，默认优化效果更好|
|**产物体积**|通常更大（冗余代码多）|通常更小（`Rollup` 更优的 `Tree Shaking`）|
|**兼容性**|原生支持低版本浏览器|默认仅支持现代浏览器，需 `@vitejs/plugin-legacy` 兼容低版本|
<!-- prettier-ignore-end -->

## 四、其他关键区别

1. **配置复杂度**

- `Webpack`：配置及其灵活但复杂，需要手动配置 `entry`、`output`、`loader`、`plugin`、`optimization` 等。
- `Vite`：遵循“约定大于配置”，内置对 `TypeScript`、`JSX`、`CSS`、`JSON`、静态资源等支持，开箱即用，配置量减少 `70%` 以上。

2. **依赖处理**

- `Webpack`：打包时统一处理所有依赖，支持 `CommonJS`、`AMD`、`UMD`、`ESM` 等所有模块规范。
- `Vite`：
  - 开发时：用 `esbuild` 预构建第三方依赖（将 `CommonJS/UMD` 转为 `ESM`，合并小模块减少请求）
  - 生产时：`Rollup` 处理所有依赖，纯 `ESM` 输出
  - 对非 `ESM` 规范的依赖兼容性稍差，部分老旧库可能需要额外配置。

3. **生态系统**

- `Webpack`：生态及其成熟，拥有海量 `loader` 和 `plugin`，几乎能满足任何构建需求。
- `Vite`：生态发展迅猛，主流框架（`React`、`Vue`、`Svelte`）均官方支持，但小众场景的插件数量仍小于 `Webpack`。

4. **多页面应用支持**

- `Webpack`：需要手动配置多个 `entry` 和 `HtmlWebpackPlugin` 。
- `Vite`：原生支持多页面，只需在 `pages` 目录下创建对应 `HTML` 文件即可。
